# E. Феодальная реформа

## Описание задачи

Феодальное государство представлено в виде подвешенного дерева из `n` городов (вершин). Вершина `0` — это столица, а для каждого другого города известен город, которому он непосредственно подчиняется.

В этом государстве широко распространены трансферы из феодального бюджета — одни города могут получать дотации, а другие платить дополнительные налоги. Для каждого города известен баланс с феодальным бюджетом (может быть как положительным, так и отрицательным). С порочной практикой феодальных трансферов было решено покончить, но не всё так просто.

## Механизм реформы

Поскольку государство феодальное, руководитель не может просто изменить налоги или дотации в определённом городе, ведь "вассал моего вассала — не мой вассал". Однако можно издать эдикт, который изменит баланс на `+1` или `-1` для всех городов на пути от столицы до выбранного города.

Таким образом, можно выбрать любой путь в дереве подчинения от вершины `0` до любой другой вершины и с помощью эдикта изменить баланс на `1` во всех вершинах этого пути.

## Цель

Сделать баланс во всех городах равным `0`, используя наименьшее возможное количество изданных эдиктов.

## Формат входных данных

- Первая строка: целое число `n` — количество вершин в дереве (`1 ≤ n ≤ 100,000`)
- Следующие `n-1` строк: по одному целому числу `p_i` — номер вершины, являющейся предком вершины `i` (`0 ≤ p_i < n`, `1 ≤ i < n`)
- Последняя строка: `n` чисел — исходные значения балансов в вершинах (`|a_i| ≤ 100,000`)

## Формат выходных данных

- Одно число — минимальное количество эдиктов, необходимых для проведения реформы
