# C. Очередь из кандидатов

## Условие задачи

За забором завода по производству программ выстроилась очередь из кандидатов, желающих устроиться на работу. Исходно в очереди стоит `n` человек. Люди в очереди пронумерованы по порядку, начиная с нуля. Номер человека в очереди равен числу людей, которое стоит в очереди перед ним.

В конец очереди могут вставать новые кандидаты, а из начала очереди кандидаты могут проходить на собеседование.

**Профессионализм** `i`-го человека — целое неотрицательное число `aᵢ`. Во время стояния в очереди профессионализм кандидата не меняется.

**Подходящий кандидат** — кандидат, профессионализм которого не меньше `x`. В противном случае кандидат считается неподходящим.

Иногда к HR Васе приходит коллега Маша и спрашивает, сколько подходящих кандидатов среди первых `k` человек в очереди.

## Задача

Реализовать программу, которая правильно отвечает на запросы Маши.

## Формат ввода

### Первая строка
- `n` — начальное количество кандидатов в очереди (1 ≤ n ≤ 100000)
- `x` — нижняя граница профессионализма (0 ≤ x ≤ 10⁹)

### Вторая строка
- `aᵢ` — профессионализм людей в очереди (0 ≤ aᵢ ≤ 10⁹)

### Третья строка
- `m` — количество событий (1 ≤ m ≤ 100000)

### Следующие m строк

1. **Добавление кандидата**
- В конец очереди приходит человек с профессионализмом `a` (0 ≤ a ≤ 10⁹)
