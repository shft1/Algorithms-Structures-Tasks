# Тестовые тренировки

В 21879 сезоне тренировок вместо задач стали использоваться вопросы с вариантами ответа. В тесте было `n` вопросов, на каждый вопрос предлагалось 4 варианта ответа: `A`, `B`, `C` и `D`. Нужно было выбрать один правильный ответ.

Две работы считаются **похожими**, если:

- в каждой из них больше половины правильных ответов совпадают с ответами в другой работе;
- в каждой из них больше половины неправильных ответов совпадают с ответами в другой работе.

Необходимо найти все пары похожих работ.

## Формат ввода

Первая строка содержит число `n` — количество вопросов в тесте (`1 <= n <= 100`).

Вторая строка содержит правильные ответы: строку длины `n`, состоящую из символов `A`, `B`, `C` и `D`. `i`-й символ строки равен правильному ответу на `i`-й вопрос.

Третья строка содержит число `m` — количество студентов, писавших тест (`1 <= m <= 100`).

Далее следуют `m` строк, содержащих ответы студентов в том же формате.

## Формат вывода

В первой строке выведите количество пар похожих работ. В следующих строках выведите все пары похожих работ в любом порядке.

Работы пронумерованы числами от `1` до `m` в том порядке, в котором они заданы во входных данных. Элементы внутри пары тоже можно выводить в любом порядке.

## Пример 1

### Ввод

```text
3
AAA
4
ABA
ABA
CBA
CAA
```

### Вывод

```text
1
1 2
```

## Пример 2

### Ввод

```text
6
ABCDAB
3
ABCCCC
BBCDCC
ACCDCC
```

### Вывод

```text
3
1 2
1 3
2 3
```
