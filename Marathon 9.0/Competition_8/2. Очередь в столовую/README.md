# 2. Очередь в столовую

## Условие

Женя пришёл в столовую и увидел, что в очереди стоит `n` человек.  
У каждого человека есть:

- **Предел терпения** `t_i` — сколько секунд он готов ждать своей очереди.
- **Время заказа** `k_i` — сколько секунд он делает заказ.

Время ожидания человека — это **суммарное время заказа всех людей перед ним**.  
Гарантируется, что изначально (до добавления Жени) для каждого человека время ожидания не превышает его предел терпения.

Женя — дружелюбный, знает всех в очереди и может встать **в любое место**.  
Он хочет встать **как можно ближе к началу очереди**, но так, чтобы из-за него **ни у кого время ожидания не превысило предел терпения**.  
У самого Жени терпение **бесконечно**.

## Формат ввода

Первая строка:  
`cases` (1 ≤ `cases` ≤ 10 000) — количество наборов входных данных.

Далее для каждого набора:

- Первая строка: `n` и `d` (1 ≤ `n` ≤ 10⁵, 1 ≤ `d` ≤ 10⁹)  
  — количество людей в очереди и длительность заказа Жени.
- Следующие `n` строк:  
  `t_i` `k_i` (1 ≤ `t_i`, `k_i` ≤ 10⁹)  
  — предел терпения и время заказа i-го человека.

**Сумма `n` по всем наборам** ≤ 10⁶.

## Формат вывода

Для каждого набора выведите одно целое число — **минимальное место** (начиная с 1), на которое может встать Женя.
